ECHS High Altitude Balloon Project

Introduction

The goal of our project was to design launch a high altitude weather balloon containing a scientific experiment and sensors, in order to measure atmospheric data. To do this we split into a media team, and engineering team, and a science team.

The Engineering team was tasked with building the sensors that would go up in the balloon. This involved soldering many different components together as well as writing code to collect data on the atmosphere as the balloon ascended.

Engineering Team

The engineering team was tasked with designing and creating a system of sensors which would measure atmospheric data. This involved a pressure sensor, a thermometer, and an accelerometer.

Making sure each sensor worked individually.

Each sensor was an integrated circuit. In order to get each one working, we needed to wire it to an Arduino board, and write code which allowed us to read data from the sensor, and translate it into understandable units.

Ensuring we could store the data.

Our first approach was to use the onboard Arduino memory, called EEPROM. We wrote code which would ‘write’ the temperature, acceleration, and pressure values to the EEPROM every minute or so. The primary constraint with this system was that there was only around 1mb of data, meaning we could only record one data point per minute for around 5 hours.

Because of this size restriction, we decided to use an SD card. We connected the SD card to the Arduino with a SD card reader, and downloaded packages which allowed us to create and write to files. Having more storage space allowed us to store data every 1 second. We also recorded the timestamp of when each piece of data was collected, using a built in time function of the arduino board.

Integrating the circuits.

Now that we had every circuit working individually and a way to store the data, we needed a way to make sure everything worked together. To do this, we created a schematic of how the circuit would look with everything connected. Once we created this, we soldered the circuit onto a perfboard.

We also needed to modify the code to work together in one piece. We turned the code for reading data from each individual sensor into its own function. We then created a ‘run’ function, which called each sensor function consecutively, stored that data to the SD card, and then waited a given amount of time before repeating the process

Securing the system to the payload.

Because the balloon was going to experience turbulence in the air, as well as a sudden change in acceleration when it popped and hit the ground, we needed to ensure that all parts of the system were solidly secured to the payload. To do this we duct taped the different parts of the circuit to different parts of the payload.

Results

After parsing through the data we gathered from the flight, we realized that unfortunately, the sensors stopped giving data around 3 hours into the flight, before the balloon reached its peak altitude. This means that we did not get the most interesting part of the flight, namely when the balloon pops and begins to descend. However, the data we did gather is still insightful, and using it we compiled the graphs below:

While we didn’t have an “altimeter”, or altitude sensor, we were able to calculate altitude based on pressure. Pressure and altitude are inversely proportional, meaning as one goes up the other goes down.

As a general rule, temperature decreases 3.5 degrees F for every 1,000 feet of elevation gain. This is because the atmosphere becomes thinner. However, you can see that around 11k seconds (or 13.5k feet of elevation), the temperature begins to rise again.

Pressure is a measure of the force on an area. There is a good amount of pressure where we live (on Earth’s surface), but almost none in outer space. As the balloon moved upwards towards outer space, pressure decreased.

We were also measuring vertical acceleration, Despite the acceleration graph looking like it’s all over the place, acceleration actually remains fairly constant (see the numbers on the y axis).
